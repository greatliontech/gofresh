# Generic subjects widen wholesale; constraint-aware narrowing is unbuilt

Lands: when a corpus demonstrably loses serving precision to open-world
generic subjects (refined records widening to maximal on unrelated edits),
and only then via constraint-aware openness plus instantiation-rooted
traversal together.

## Observed

A parameterized subject reads open-world from its signature: the
dynamic-signature walk treats its type-parameter constraint as an
interface, so refinement widens to the maximal closure and observability
refuses on the open subject world - sound, and exactly what the
signature rule gives every dynamism-accepting subject. The crash fix
that established this (attributed RTA panicking on parameterized
bodies) deliberately did not add precision machinery: instantiation
rooting was built and then removed when measurement showed no
observable behavior depends on it while the open-world wall stands.

## Resolution shape

Two coupled pieces, worthless separately:
- constraint-aware openness: a type parameter whose constraint's type
  set provably excludes dynamic carriers (no interface methods, no
  channels, no unsafe reach) does not open the subject's world;
- instantiation-rooted traversal: a parameterized subject then roots
  its materialized instantiations for reachability, its origin folded
  for content (the removed machinery, recoverable from git history).

Both must pass the batch-equivalence and purity-override audits: under
a purity assertion an open-world subject serves on refined-hash
equality, so narrowed openness must never lose instantiation-reachable
content from the hash.
